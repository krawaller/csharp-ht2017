2-3. Node and NPM

workbench and pantry

2-3-1

In the course we'll rely heavily on

  • athe JavaScript runtime Node and

...so let's touch briefly on both!

2-3-2

Node is mostly associated with running JavaScript on the server, but in actuality it is a JavaScript runtime that can be run anywhere. Some differences versus the in-browser runtime is that

  • Node has no DOM
  • Node can read and write files
  • Node has its own module system
2-3-3

There are two ways to execute JS with Node. We can enter the REPL by typing node in a node terminal...

2-3-4

...or we can execute JavaScript files by typing node myfile.js which executes the JS code in myfile.js.

This of course is the most common usage of Node.

JS developers frequently use Node to run code as part of their build chain.

2-3-5

However, Node is also used for running npm, through which we can download packages from the online repository at https://www.npmjs.com/.

2-3-6

Allegedly npm is an acronym for Node Package Manager, but npm themselves deny that through consistently using lower-case characters, and through the top-left corner of their homepage:

2-3-7

We download packages by

  • turning our own code into a package through adding a package.json file
  • listing the packages we depend on in that file
  • typing npm install to download all listed dependencies to node_modules
  • using those modules in our code!
2-3-8

By having package.json as part of the project, but excluding node_modules, we get a small code repository which can still be easily inflated with dependencies.

And we can still be sure that everyone has the same dependcies.

2-6. Objects

bags of data

2-6-1

This section is an overview of objects, in case you need a refresher!

2-6-2

JavaScript has two composite data types, namely objects and arrays!

Althought technically arrays are a special kind of objects, but, never mind.

2-6-3

An object in JavaScript is a collection of key-value pairs, often called a dictionary in other languages.

The property names (keys) are always strings, and the property values can be anything.

2-6-4

We can create an object literal like this:

let myObject = {
  someProp: 42,
  anotherProp: "hello!"
};
2-6-5

There are two different ways to access a value.

We can use dot notation:

let x = myObject.someProp; // 42
2-6-6

Or we can use bracket notation:

let y = myObject["someProp"]; // 42
2-6-7

Question: The dot notation seems much smoother, so why would we ever want bracket notation?

2-6-8

Answer: That allows us to do dynamic lookups:

let propName = "someProp";
let z = myObject[propName]; // 42
2-6-9

We add a property simply by assigning to it:

let myObject.newProp = 777;
2-6-10

And we delete a property using the delete keyword:

delete myObject.newProp;
2-6-11

If we access a non-existing property we always get undefined.

myObject.bogusProp; // undefined
2-6-12

Contrary to primitives, objects are references, which means that this is false since it is two different objects:

let test = ( {a:1} === {a:1} ); // false
2-6-13

While this is true, since they point to the same object:

let x = {a:1};
let y = x;
let test = x === y; // true
2-6-14

Since objects are references they are always truthy:

if ({}){
  console.log("An empty object is truthy!"); // will be shown!
}
2-6-15

We can iterate over objects using a for-in loop:

for (let key in myObject){
  console.log(key+" has value "+myObject[key]);
}
2-6-16

Finally: object can contain other objects:

let me = {
  name: "David",
  address: {
    street: "Åsgatan 9",
    zip: 27131,
    city: "Ystad"
  }
}
2-6-17

We can access properties to any depth: 

let myZip = me.address.zip;

...although we might have to guard for undefineds:

let myZip = me.address && me.address.zip;

2-7. Arrays

Ordered lists

2-7-1

This section is an overview of arrays, in case you need a refresher!

2-7-2

A special kind of objects are arrays, which are an ordered list of values.

You could say that they are objects where the keys are always named 0, 1, 2....

2-7-3

We can create arrays using the array literal notation:

let list = ["tomatoes",42,false];
2-7-4

Since arrays are objects we access elements as before:

let firstItem = list[0]; // "tomatoes"
2-7-5

An array has a length property that tells us how many elements it contains:

list.length // 3
2-7-6

Which means I can pick the last item by doing this:

let lastItem = list[list.length-1]; // false
2-7-7

We can add a new value by calling the push method:

list.push("foobar");
list.length // 4
2-7-8

It is common to iterate using a regular for loop:

for(let i=0; i < list.length; i = i+1){
  console.log("Item "+i+" is "+list[i]);
}
2-7-9

There are a whole bunch of array methods - see the MDN reference for a full list:

https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Array

2-7-10

Question: The length property was rather convenient - is there something similar on regular objects?

2-7-11

Answer: Not directly, but we can get an array of all keys for an object using the keys method of the global Object object:

let keys = Object.keys(myObject); // ["someProp","anotherProp"]

2-8. Functions

code in a can

2-8-1

This section is an overview of functions, in case you need a refresher!

2-8-2

There are two different ways to define a function in JavaScript. First as a named function:

function echoer(word,times){
  let echo = "";
  for(let i = 0; i < times; i = i + 1){
    echo = echo + word;
  }
  return echo;
}
2-8-3

Or as an anonymous function which we can assign to a variable:

let echoer = function(word,times){
  let echo = "";
  for(let i = 0; i < times; i = i + 1){
    echo = echo + word + " ";
  }
  return echo;
}

Mostly it doesn't really matter which way you choose.

2-8-4

As the example showed, we use return something to return a value. If you just do return;, the returned value is undefined, which it also is if you don't use return at all.

2-8-5

We execute a function by using parens, and listing eventual parameters within those:

let withEcho = echoer("hello",3); // "hello hello hello "
2-8-6

Question: What is the difference between this:

let x = func;

and this:

let x = func();
2-8-7

Answer: Well:

  • In the first example, x has a reference to func. Which means that after that, doing x() and func() would yield the same thing.

  • In the second example, x is whatever func returned.

2-8-8

A final observation - There is nothing stopping us from having functions on objects:

myObject.beingAnnoying = function(){
  console.log("SPAM!");
};
myObject.beingAnnoying(); // "SPAM!"

When a function is a property on an object it is called a method of that object.

2-8-9

And to end on a meta note; there is also nothing stopping us from having objects (or anything else) on functions, since functions are objects.

let func = function(){};
func.prop = {hello:"world!"};
console.log(func.prop.hello); // "world!"

2-10. Functional programming

Jumping down the rabbit hole

2-10-1

JavaScript is a functional language, which modern JS makes frequent use of.

Let's just do a quick sanity check to make sure we're all on the same page!

2-10-2

So - since functions are first class citizens, we can send them around just like any value.

Which also means that a function can take, and/or return, other functions! Such a function is called a higher order function.

2-10-3

As a contrived example, say we have this function:

let spam = function(){
  console.log("SPAM!");
}
2-10-4

And then we have this higher order function:

function repeater(func,times){
  for(let i = 0; i < times; i = i + 1){
    func();
  }
}
2-10-5

If we invoke repeater like this:

repeater(spam,3);

We would see this in the console:

"SPAM!"
"SPAM!"
"SPAM!"
2-10-6

To show we can also return new functions, take a look at this beauty:

function multiplier(func,times){
  return function(){
    for(let i = 0; i < times; i = i + 1){
      func();
    }
  };
}
2-10-7

Did you see the difference? multiplier doesn't execute the parameter function, but returns a new function!

let tripleSpam = multiplier(spam,3);

If we execute the returned function we get the triple spam:

tripleSpam(); // SPAM! SPAM! SPAM!
2-10-8

Functional programming is a really powerful tool, and something that is likely to get you hooked once you have learned it. We warmly encourage you to explore the subject!

2-10-9

Note: the functional paradigm is about much more than just passing functions around, we've really just scratched the surface here.

2-11. map, filter and reduce

the cornerstones

2-11-1

For applied functional programming, there are three list-operating tools that have a very central position:

  • aThe map method
  • bThe filter method
  • cThe reduce method

Let's take a look at these!

2-11-2

First, map. It takes a list and an iterator function as arguments, and returns a new list.

2-11-3

The iterator function that we pass in is called with each element and the index of that element, and it should return a new element to be used instead:

2-11-4

Here's an example. What will this call return?

let list = ["David", "Carl", "Eric"];
let newList = list.map(function(elem){
  return elem+"y Mc"+elem+"face";
});
2-11-5

Yup, newList will now equal this:

["Davidy McDavidface","Carly McCarlFace","Ericy McEricface"]
2-11-6

In essence, .map will create a new list of the same length, where each element is individually transformed by the iterator.

2-11-7

Next up is .filter, which is probably the easiest to understand. It has the exact same signature as .map:

2-11-8

The iterator is similar too, except it returns a boolean saying whether or not the element should go into the new array.

2-11-9

Now for .reduce! It is more powerful, but also harder to grasp.

2-11-10

Similar to .map it takes a list and an iterator, but also a seed value. Unlike .map it doesn't necessarily return a list, instead it can return any value.

2-11-11

The iterator function is called with each element and the current value (sometimes called memory or accumulator), and returns a new value.

2-11-12

The seed that we passed in to .reduce is used as the current value for the first iterator call.

The result of the .reduce call is whatever is returned from the last iterator call.

2-11-13

In other words, it looks like this (for a 3-item list):

2-11-14

Here's a code example. What does it return?

let list = ["David", "Carl", "Eric"];
let something = list.reduce(function(mem, elem){
  mem[elem.length] = mem[elem.length] || 0;
  mem[elem.length]++;
  return mem;
}, {})
2-11-15

it returns an object which counts how many words of each length the list contains:

{
  4: 2,
  5: 1 
}
2-11-16

As previously stated, .reduce is hard to grasp but immensely powerful.

A quick proof of that is that it can be used to implement both .map and .filter. So .reduce is really all we need!

3-1-3

3-3. Git and Github

The tools of the trade

3-3-1

Chances are very slim that you'll be able to do modern web work without using...

  • Git for version controlling and collaboration
  • Github for an online-hosted master Git repo
3-3-2

Probably you're already up and running (?), but here's a condenced guide for common patterns:

  • aFirst-time setup
  • bThe idea of a commit
  • cThe commit flow
  • dExcluding files
  • eSeeing what changed
  • fA log of changes
  • gCentral master repo
3-3-3

If this is your first time with Git, you will want to start by configuring Git to recognize your name and email address.

git config --global user.name "Your Name Here"
git config --global user.email "your_email@youremail.com"
3-3-4

Git centers around the idea of a commit. We make a commit for each change you wish to record in the system

Some version control systems keep track of history per file. Git versions the whole tree.
3-3-5

A commit consists of:

  • A snapshot of all the files
  • A commit message
  • An author
  • A date
3-3-6

In Git, committing is a two-step process

3-3-7

The add command places a file in the staging area Below is given that the README file exists

$ git add README.md

The commit command then takes the contents of the staging area, and creates a commit

$ git commit

[master (root-commit) beb9dfd] Add a README.
 1 files changed, 1 insertions(+), 0 deletions(-)
 create mode 100644 README
3-3-8

For existing files, add will place the changes to that file into the staging area, so you can commit changes with two commands:

$ git add README
$ git commit

(A shorthand for these steps: git commit README)

3-3-10

We want to avoid tracking generated files with Git. Let's say these are created by our build step:

    # On branch master
# Untracked files:
#
#       main.exe
#       main.obj
#       util.obj
3-3-11

The solution avoid tracking them is to add a .gitignore file

Content for .gitignore

*.exe
*.obj

Then commit the file

$ git add .gitignore
$ git commit
3-3-12

The .gitignore file won't be of immediate use, but we'll employ it in the next chapter.

3-3-13

While git status gives you an overview of your changes, git diff shows what changed.

$ git diff

diff --git a/src/util.c b/src/util.c
index 9fe5927..e826a45 100644
--- a/src/util.c
+++ b/src/util.c
@@ -1,7 +1,7 @@
 #include <stdio.h>

-void print_ten_times(char *msg) {
+void print_n_times(char *msg, int n) {
     int i;
-    for (i = 0; i < 10; i++)
+    for (i = 0; i < n; i++)
         printf(msg);
}
3-3-14

This is so useful that VSC has a tab for it:

3-3-15

Typing git log shows a log of all the commits, most recent commit first

    commit e20962ebed7b0288922320f217a6a3ab9371727c

3-3-16

The log command has more options than you can possibly imagine; one fairly useful one is:

$ git log --oneline

To learn about (literally) dozens more, see:

3-3-17

The most common use of remotes is to set up a central repository

It's the central repository because everyone agrees it is. To Git, it's just another copy.
3-3-18

There are a range of hosted Git services out there, which can host your central repository for you

GitHub is the largest today, offering free hosting for public repositories (used by thousands of open source projects) and private hosting for individuals and organizations

3-3-19

Creating an account is free, and easy.

If you haven't already got one, simply go to https://github.com/ and follow the directions!

3-3-20

After creating a repository on GitHub, we need to tell our local Git repository about it by adding it as a remote:

$ git remote add origin https://github.com/<user>/<repo>.git
3-3-21

Since you can have multiple remotes, you have to specify a name as well as the address

The convention used by almost all Git users is to give the central repository remote the name origin, but there's no technical magic to the name.

3-3-22

Pushing is taking commits we have locally and copying them to a remote.

Our first push should use the -u flag:

$ git push -u origin master

After that you can simply push:

$ git push

3-5. DOM nodes

node juggling

3-5-1

Let's refresh...

  • ahow to get references to DOM nodes
  • bhow to manipulate a DOM node
  • chow to create DOM nodes
3-5-2

We can get a reference to a DOM node in a bunch of different ways. Some methods return a single element:

let node = document.body;
let node = document.getElementById("someId");
let node = document.querySelector("div > p:first-child");
3-5-3

..while others give us a list of matching nodes:

let nodes = document.getElementsByTagName("div");
let nodes = document.getElementsByClassName("someClass");
let nodes = document.querySelectorAll("div > p:first-child");
3-5-4

From a node reference we can access the element above that element...

node.parent

...or the elements below:

node.children
3-5-5

Once we have a node reference there's a huge number of things we can do with it! The DOM API is enormous.

But here we'll just show a few useful things!

3-5-6

We can add a class...

node.classList.add("someClass");

...or remove it...

node.classList.remove("someClass");

...or test if it has a certain class:

node.classList.contains("someClass") // true or false
3-5-7

We can change the text content of a node...

node.textContent = "Hello world!";

...or HTML content:

node.textContent = "<strong>Hello world!</strong>";

In both cases we erase eventual previous content.

3-5-8

What about attributes on elements?

<div data-ref="Ek#AfZW2">

We can query them...

node.getAttribute("data-ref"); // "Ek#AfZW2"

...and set them:

node.setAttribute("data-ref","someNewRef");
3-5-9

Finally, let's see how we create DOM nodes!

let newNode = document.createElement("div");

Just instantiating a node like this doesn't do anything - it isn't connected to the document.

3-5-10

In order to accomplish that we need to append or prepend the node to a parent node:

let existingNode = document.getElementById("someId");
existingNode.appendChild(newNode);

There is of course a corresponding .prependChild too.

3-7. DOM event refresher

Gotta catch 'em all

3-7-1

Another thing we can do with a node reference is to make code run when things happen to it.

The API for this is simple:

node.addEventListener("nameOfEvent", functionToRun);

The function we pass in is called an event handler.

3-7-2

Here's a simple example:

function goBoom(){
  alert("You pushed the button. BOOM goes the world!");
}

node.addEventListener("click", goBoom);
3-7-3

Remember to pass in a reference to the event handler, as opposed to executing it!

// this is wrong!
node.addEventListener("click", goBoom());
3-7-4

It is quite common to define the event handler in place as an anonymous function:

node.addEventListener("click", function(){
  alert("You pushed the button. BOOM goes the world!");
});
3-7-5

The event handler is called with an event object, which might be useful for some events:

inputFieldNode.addEventListener("keyup", function(e){
  if (e.key === "Enter" || e.keyCode === 13){
    submitForm();
  }
});

3-9. jQuery refresher

What the fuzz is about

3-9-1

You've likely come across jQuery, a library that makes the DOM easier to work with.

3-9-2

Let's see how some common tasks are done with jQuery:

  • anode references
  • bnode manipulation
  • cnode creation
  • devent handling
  • eevent delegation
3-9-3

We get node references by passing a CSS selector to the jQuery function:

// returns a jQuery collection of all elements matching that selector
let collection = jQuery(".someClass");
3-9-4

You'll rarely see jQuery being used though, instead we work with the $ alias:

let collection = $(".someClass");
3-9-5

Here's some common manipulations:

collection.html("<strong>verboten</strong>");
collection.text("unavailable");
collection.addClass("hide");
collection.attr("data-ref","Ek#AfZW2");
collection.empty(); // killing all children
3-9-6

Note that unlike the DOM, jQuery doesn't have different methods for getting 1 or many nodes.

We can manipulate the resulting collection as if it was one node, and jQuery will do the changes to all nodes.

3-9-7

That means that this regular JS...

let nodes = document.querySelectorAll(".french");
for(let node in nodes){
  node.innerHTML = "<em>click to translate</em>";
}
3-9-8

...can be done like this in jQuery:

let collection = $(".french");
collection.html("<em>click to translate</em>");

Commonly we skip the variable and chain instead:

$(".french").html("<em>click to translate</em>");
3-9-9

Now to see how we create new elements with jQuery!

Normally, if you pass a string to jQuery, it'll treat it as a CSS selector and collect all matching element:

let collection = $(".blogpost > p:first-child");
3-9-10

However, if we pass in a string containing HTML code, jQuery will instead create the matching element!

let newCollection = $("<span>");
3-9-11

This HTML code can contain multiple elements as well as attributes:

let newCollection = $(`
  <div class="container">
    <h2>Hello world!</h2>
    <p>All this was created in a multiline string sent to jQuery!</p>
  </div>
`);
3-9-12

Then we can append the new node:

$(selectorForParent).append(newCollection);

There's also a reversed syntax:

newCollection.appendTo(selectorForParent);
3-9-13

Of course we could have simply used the .html method:

$(selectorForParent).html(stringWithNewHtml);
3-9-14

Event listeners are added using the on method:

$("#submitbtn").on("click",handleSubmission);

Although it's common to use the event-specific shorthands:

$("#submitbtn").click(handleSubmission);
3-9-15

jQuery also makes it very easy to do event delegation.

This is a powerful pattern that lets us listen to events on many elements - which might not even exist yet - using only a single event listener!

3-9-16

We accomplish this by listening for the event on an ancestor element, and then letting the event bubble up to there from the element we're actually interested in.

3-9-17

Consider an app with this UI:

<ui id="options">
  <li data-option-id="gowest">Go west</li>
  <li data-option-id="opendoor">Open the door</li>
  <li data-option-id="turnback">Turn back</li>
</ul>

The contents of the #options list will change as the user clicks around.

3-9-18

Instead of adding a listener everytime we create an option, we can have a single listener on the list which we filter to only handle clicks on options:

In jQuery we do this by passing a filtering selector after the event name to .on:

$("#options").on("click", "li", function(e){
  let opt = $(e.target).attr("data-option-id");
  gameEngine.executeOption(opt);
});
